A Strong Baseline for Query Efficient Attacks in a Black Box Setting

Posted on 2021-10-28 Valine:

2021-EMNLP　Black Box(其实用了score，不算严格黑盒); Query Efficient; untarget attack;

Tasks: text classification and entailment

Model: Bert, WordCNN, WordLSTM

Dataset: IMDB, Yelp, MNLI

这篇文章的名字取得有一点嚣张，a strong baseline，搞得好像后面的文章不引用就不行一样。

主要方法是引入了Attention机制和LSH对词的重要度进行了排序。

现在从写作的角度来看，主要是想知道这篇文章为什么可以发顶会。

第一点是，写的深入，将所想的东西都表达出来了。比如把现有的黑盒攻击分为两个思考方向，一个是搜索空间，一个是搜索方法，再比如，删掉一个词进行排序的方法为什么不好，其实这些都是我们在实验过程中有思考的问题，但是拿出来写文章的时候就完全不知道写什么，现在看来，之前的文章就写的很浅。

第二点是，文章的卖点，这篇文章引入了排序机制是一个卖点，攻击成功率和查询次数两个指标也是一个卖点。现在的问题就是如何把自己简单的方法描述的很高级，然后阐述这么做为什么效果会好，说明我们的方法不是偶然发现的一个方法，而是经过思考和对比的。

真的觉得自己做的东西效果不比别人差，主要还是writing的功力不够，继续polish吧。
